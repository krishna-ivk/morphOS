# P1 Universal Delivery Terminology Spec

## Origin & Influence
- **Source Influence**: Universal delivery naming work, Paperclip-inspired operator language
- **Local Owner(s)**: `sky-force-command-centre-live`, `morphOS`

## Core Concept
The control plane for `morphOS` should read like standard Delivery Software (e.g. GitHub Actions, Vercel, Linear), not an internal AI protocol console. If terms leak from the durable execution engine (`wavezync/durable`) into the UI, operator adoption will fail due to cognitive load.

## Concrete Terminology Mapping

| Internal Architecture Term (Hidden) | Universal Terminology (UI / Operator Visible) | Context |
| :--- | :--- | :--- |
| `DurableExecutionRequest` | **Run** or **Job** | The overarching execution container. |
| `ExecutionEnvelope` | **Step** | A specific sandboxed execution command. |
| `ApprovalDecision` target | **Review Request** | When a workflow pauses for an Admin. |
| `ContextRef` / `MemoryNode` | **Artifacts** | Files or knowledge generated by an agent. |
| `Agent` / `Archetype` | **Copilot** or **Worker** | The entity performing the action. |

## Multi-Repo Responsibilities
- `skyforce-core` contracts MUST use the **Internal Architecture** terms (e.g., `interface ExecutionReceipt`) to ensure precision across microservices.
- `sky-force-command-centre-live` is exclusively responsible for translating these dense TS interfaces into the **Universal Terminology** strings presented to the Workspace Admin.
- `morphOS` owns the canonical mapping and the doctrine behind it.

### `skyforce-command-centre`

Should use the universal terms by default in UI text.

### `skyforce-core`

May preserve internal type names while also carrying operator labels where useful.

### `skyforce-symphony` and `skyforce-harness`

Should avoid inventing conflicting human-facing language in logs, summaries, or CLI output.

## P1 Success Condition

This spec is successful when:

1. the same delivery object is described with the same operator term across surfaces
2. the `P0` demo artifacts can be explained without internal-only jargon
3. internal type precision is preserved without forcing that language onto operators

## Bottom Line

Universal terminology is the naming layer that makes the factory operable by humans, not just understandable to runtime engineers.
