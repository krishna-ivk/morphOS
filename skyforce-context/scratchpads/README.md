# Agent Scratchpads

This directory contains live "scratchpads" where agents document their granular, step-by-step progress before it is finalized and reported to the tracker (Linear).

## Purpose
- **Transparency**: Allows other agents to see active work-in-progress.
- **Context Preservation**: Prevents loss of state if a session is interrupted.
- **Handoff**: Serves as the raw data for generating final summaries.

## Rules
1. Each agent session should maintain a file here: `scratchpads/<agent>-<issue-id>.md`.
2. Use timestamps for all entries.
3. Link to relevant artifacts or build logs.
